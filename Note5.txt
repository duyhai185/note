Sau khi các bạn mkt commit lên, 
	mình sẽ pull về máy
	kiểm tra xem file md đó có thiếu hay sai cú pháp gì không,
	chạy preview xem ổn chưa, rồi mới push lên

nhớ check các vấn đề
1.tất cả tài liệu đều viết bằng cú pháp markdown
	bảng phải viết dạng markdown
	tham khảo cấu trúc:
	https://www.markdownguide.org/cheat-sheet/

	và chuyển html to markdown
	https://codebeautify.org/html-to-markdown
	https://htmlmarkdown.com/

	kiểm tra các cấu trúc table, code, ảnh, text heading

2. kiểm tra code php, js, css xem có đúng chuẩn của Wordpress chưa
	https://developer.wordpress.org/coding-standards/wordpress-coding-standards/
	https://kb.elightup.com/chuan-viet-code-php/
	và phải dùng tab vào chứ không dùng cách, dùng cách cách thụt dòng là sai, không chuyên nghiệp
	code xem đóng mở đã đủ chưa, có khi thiếu,check code kĩ

3. Code để trong file phải systax highlight
	trước ``` phải để ngôn ngữ của đoạn code đó, ví dụ ```php hoặc ```js, ```css

4. Code chỉ cần promot AIO thôi không cần promot các extension riêng nữa

5. chạy xem kiểm tra lần cuối
	- nếu đang ở file đó, Bấm Ctrl + Shift + V để xem preview hoặc vào trang https://markdownlivepreview.com/
	- nếu muốn chạy xem ở 1 repo:
	npm install (nếu chưa chạy)
	npm run build
	npm run start
